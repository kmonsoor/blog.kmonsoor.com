Some are cutting-edge, some are cutting edge. Some born to boot-up IoT devices, some are to pull up heavy-graphics.</p> <p>That's the fun (albeit, power) of Linux.</p> <p><img alt="An uber, cool image by Ash Edmonds" src="https://i.imgur.com/3lUdFA4.jpg"></p> <blockquote> <p>Photo by <a href="https://unsplash.com/@badashproducts">Ash Edmonds</a> on Unsplash</p> </blockquote> <p>CentOS 7 is a powerful and stable distro which runs on thousands (probably, millions) production-grade servers.<br> In stability, it's a beast. However, it don't ship with Python 3, by default. You can install it via EPEL repository, or this below simple steps.</p> <p>Also, take a note. The Python <strong>2</strong> comes with the system, which is probably 2.7.5, do <strong>NOT</strong> mess with it. Many system components rely on that specific version. If you need later version of 2, use virtualenv or pipenv.</p> <h3>Prepare your system</h3> <p>Start with installing pre-requisite utilities for compilation and development support.</p> <div class="highlight"><pre><span></span>$ sudo yum update <span class="o">&amp;&amp;</span> sudo yum groupinstall -y <span class="s2">&quot;development tools&quot;</span> $ sudo yum install -y zlib-devel bzip2-devel openssl-devel ncurses-devel <span class="se">\</span> sqlite-devel readline-devel tk-devel gdbm-devel <span class="se">\</span> db4-devel libpcap-devel xz-devel expat-devel </pre></div> <h3>Download latest Python source code from Python.org</h3> <div class="highlight"><pre><span></span>$ wget https://www.python.org/ftp/python/3.6.6/Python-3.6.6.tar.xz $ tar xf Python-3.6.6.tar.xz $ <span class="nb">cd</span> Python-3.6.6 </pre></div> <h3>Enable performance optimizations (optional, but highly recommended)</h3> <div class="highlight"><pre><span></span>$ ./configure --prefix<span class="o">=</span>/usr/local --enable-shared <span class="nv">LDFLAGS</span><span class="o">=</span><span class="s2">&quot;-Wl,-rpath /usr/local/lib&quot;</span> $ ./configure --enable-optimizations </pre></div> <h3>Build and install</h3> <div class="highlight"><pre><span></span>$ make $ sudo make altinstall </pre></div> <p>Now, Python 3.6.6 is ready to be used in your system; I also stuck with same problem; researched many forums, and here is the gist. Find out how.</p><p>Stuck with your small virtual disk partition for VirtualBox? You created for some testing &amp; practicing, now it needs space. I also stuck with same problem; researched many forums, and here is the gist.</p> <p>[Note: This procedure won't work for VM with snapshots. So, please take note about your system beforehand.]</p> <ul> <li>Download <a href="http://gparted.org/download.php">Gparted</a>, disk manager for Linux. It'll come as an ISO file.</li> <li>Boot into your guest OS, check, with df command, which partition you need to grow. Take note of that specific mounted HD, such as /dev/hdb/sda3</li> <li>Take backup of your existing data, as I WILL NOT TAKE ANY RESPONSIBILITY IF YOU DAMAGE YOUR DATA.</li> <li>In host OS, Goto the command prompt by running, cmd.exe</li> <li>Navigate to the Virtualbox installation folder</li> <li>Execute command:</li> </ul> <p><code>VBoxManage modifyhd X:\yourVM_DiskPath_InHost\yourVM_Disk.vdi --resize 40960</code></p> <p>you will see progress as below:</p> <p><img alt="10% ... 20% .. .. 100%" src="http://i.imgur.com/iYORelg.png"></p> <p>If you fail with message like</p> <blockquote> <p>"VBoxManage.exe: error: Resize hard disk operation for this format is not implemented yet!"</p> </blockquote> <p>you need to use this tool <a href="https://forums.virtualbox.org/download/file.php?id=7579">CloneVDI.exe</a> from this VirtualBox forum-post. If using this tool, remember to check "Increase virtual drive size to " with your desired size.</p> <p><img alt="VDI_tool" src="http://i.imgur.com/YB49ZVk.png"></p> <ul> <li>Now, Load the ISO file to the CD/DVD drive of your virtual Linux.</li> </ul> <p><img alt="GParted.iso loaded on IDE Primary Master" src="http://i.imgur.com/UKyPkl8.png"></p> <ul> <li>Boot the Guest OS; from the boot menu, select CD/DVD drive, then it will be booted in GParted tiny OS. Now the PartitionManager tool will come up automatically, like below:</li> </ul> <p><img alt="Gparted partition-manager" src="http://i.imgur.com/aK9kAtK.jpg"></p> <ul> <li>Now "shrink/Grow" or "Move" your desired partion, but be careful. Don't rename any of them. Be careful about your data.</li> <li>Don't worry, unless you clicked Apply, nothing has happened really. When you are done with moving and resizing, you have to click <code>Apply</code> to commit the change</li> </ul> <p><img alt="just before applying" src="http://i.imgur.com/wUMCHVQ.jpg"></p> <ul> <li>when completed, close Gparted, and then shutdown the OS</li> <li>Unload the ISO aka virtual CD from virtual OS</li> <li>Boot into the virtual OS again</li> <li>Now, check that if the desired partition grew or not by df command</li> </ul> <p>Thanks for visiting my blog.
